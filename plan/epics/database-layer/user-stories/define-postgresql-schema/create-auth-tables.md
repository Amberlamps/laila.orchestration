## Description

Define the authentication tables required by Better Auth in the Drizzle ORM schema. Better Auth manages user authentication (Google OAuth) and session management, requiring specific table structures for users, sessions, and OAuth accounts.

The key design decision in this system is the **tenant_id = user_id pattern**: each user IS a tenant. The `user.id` field serves as the `tenant_id` for all tenant-scoped data. This simplifies multi-tenancy by eliminating a separate tenants table and making data isolation straightforward.

- [ ] `accounts` table is defined with columns: `id` (UUID, primary key), `userId` (UUID, foreign key to users), `accountId` (text), `providerId` (text), `accessToken` (text, nullable), `refreshToken` (text, nullable), `accessTokenExpiresAt` (timestamp, nullable), `refreshTokenExpiresAt` (timestamp, nullable), `scope` (text, nullable), `idToken` (text, nullable), `createdAt` (timestamp), `updatedAt` (timestamp)
- [ ] Foreign key relationships are defined: `sessions.userId -> users.id`, `accounts.userId -> users.id`
- [ ] Cascade delete is configured: deleting a user cascades to sessions and accounts
- [ ] Indexes are defined: `users.email` (unique), `sessions.token` (unique), `sessions.userId`, `accounts.userId`, composite `accounts(providerId, accountId)` (unique)
- [ ] All tables use UUID primary keys generated by the application (not auto-increment)
- [ ] A code comment at the top of the file explains the tenant_id = user_id pattern
- [ ] The `users` table schema is exported as `usersTable` for use as foreign key references in other schema files

## Technical Notes

- Better Auth table schema in Drizzle:

  ```typescript
  // packages/database/src/schema/auth.ts
  // Authentication tables for Better Auth (Google OAuth)
  // IMPORTANT: tenant_id = user.id — each user is their own tenant
  // All tenant-scoped tables reference users.id as their tenant_id foreign key
  import { pgTable, uuid, text, boolean, timestamp, uniqueIndex, index } from 'drizzle-orm/pg-core';

  export const usersTable = pgTable(
    'users',
    {
      id: uuid('id').primaryKey().defaultRandom(),
      name: text('name').notNull(),
      email: text('email').notNull(),
      emailVerified: boolean('email_verified').notNull().default(false),
      image: text('image'),
      createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
      updatedAt: timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
    },
    (table) => ({
      emailUniqueIdx: uniqueIndex('users_email_unique_idx').on(table.email),
    }),
  );
  ```

- Better Auth has specific expectations about column names — verify the exact column naming convention required by the Better Auth adapter (snake_case vs. camelCase)
- The `users.id` serves dual purpose: authentication identity AND tenant isolation key
- Consider using `$defaultFn(() => crypto.randomUUID())` for UUID generation if `defaultRandom()` is not available
- Timestamps should use `withTimezone: true` for correct handling across time zones
- Export table references (e.g., `usersTable`) as named exports so other schema files can reference them for foreign keys
