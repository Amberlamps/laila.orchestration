## Description

Configure the `openapi-fetch` client to provide type-safe API communication between the Next.js frontend and the API backend. The client wraps the generated TypeScript types from the `@laila/api-spec` shared package to ensure all API calls are type-checked at compile time.

### Client Configuration

```typescript
// apps/web/src/lib/api-client.ts
// Type-safe API client using openapi-fetch with generated types from @laila/api-spec.
// Automatically includes auth credentials and handles common error responses.
import createClient from "openapi-fetch";
import type { paths } from "@laila/api-spec";

// Create the openapi-fetch client with the API base URL.
// The base URL points to the Next.js API routes in development,
// or the deployed API in production.
export const apiClient = createClient<paths>({
  baseUrl: process.env.NEXT_PUBLIC_API_BASE_URL ?? "/api",
  // Include session cookies with every request.
  // Better Auth sets an HttpOnly cookie that the browser sends automatically.
  credentials: "include",
});

// Add a middleware to handle common error responses globally.
// Individual query hooks can still handle errors specifically.
apiClient.use({
  async onResponse({ response }) {
    if (response.status === 401) {
      // Session expired — trigger re-authentication.
      // Use a module-level flag to prevent multiple simultaneous redirects.
      handleUnauthorized();
      return response;
    }

    if (response.status === 403) {
      // User does not have access to this resource.
      // Let the calling code handle this (usually shows 403 error page).
      return response;
    }

    if (response.status >= 500) {
      // Server error — log for debugging, let calling code handle UI.
      console.error(`API Error ${response.status}:`, response.url);
      return response;
    }

    return response;
  },
});

let isRedirecting = false;

function handleUnauthorized() {
  if (isRedirecting) return;
  isRedirecting = true;
  window.location.href = `/sign-in?returnUrl=${encodeURIComponent(window.location.pathname)}&reason=session_expired`;
}
```

### Usage Examples

```typescript
// Type-safe API calls — TypeScript enforces correct paths, methods, and body types.
// Responses are also fully typed based on the OpenAPI spec.

// GET /api/projects — returns typed project list
const { data, error } = await apiClient.GET("/api/projects", {
  params: {
    query: { status: "in_progress", page: 1, pageSize: 20 },
  },
});

// POST /api/projects — creates a new project with typed body
const { data, error } = await apiClient.POST("/api/projects", {
  body: {
    name: "My Project",
    description: "Project description",
    workerInactivityTimeoutMinutes: 30,
  },
});

// DELETE /api/projects/{projectId} — typed path parameter
const { data, error } = await apiClient.DELETE("/api/projects/{projectId}", {
  params: { path: { projectId: "abc123" } },
});
```

## Technical Notes

- `openapi-fetch` generates a type-safe client from OpenAPI TypeScript types. The types are generated by `openapi-typescript` from the OpenAPI spec and published in the `@laila/api-spec` shared package.
- The `credentials: "include"` option ensures the browser sends session cookies with every request, including cross-origin requests. The API server must respond with appropriate CORS headers (`Access-Control-Allow-Credentials: true`).
- The middleware system in `openapi-fetch` uses an `onResponse` hook to intercept all responses. This is where global error handling (401 redirect, logging) is implemented.
- The module-level `isRedirecting` flag prevents race conditions where multiple simultaneous API calls all return 401 and each tries to redirect. Only the first redirect is executed.
- In development, the base URL defaults to `/api` (same-origin Next.js API routes). In production, it may point to a separate API domain.
