# Project Learnings (Agent Memory)

- [pnpm]: Use `npx pnpm` in environments where corepack isn't enabled (permission issues with `corepack enable`)
- [gitignore]: Always add node_modules/ to .gitignore when initializing a monorepo ‚Äî the pre-existing .gitignore may not include it
- [eslint]: ESLint v9 flat config uses `eslint-config-prettier/flat` import path for flat config compatibility
- [monorepo]: Root package.json scripts use `pnpm -r run <script>` for recursive workspace execution
- [eslint]: Wrap numbers in `String()` inside template literals ‚Äî `@typescript-eslint/restrict-template-expressions` forbids bare number interpolation
- [eslint]: Avoid single-use type parameters in generics (`@typescript-eslint/no-unnecessary-type-parameters`) ‚Äî use `unknown` return + cast at call site instead of `<T>() => T`
- [eslint]: Don't use rest-destructuring to omit fields (e.g. `const { foo: _foo, ...rest } = obj`) ‚Äî `@typescript-eslint/no-unused-vars` flags the discarded var. Instead, construct the desired object explicitly without the omitted field
- [deps]: Always add runtime dependencies to package.json before importing them ‚Äî missing `react-hook-form`, `@hookform/resolvers`, or `date-fns` causes cascading `import-x/no-unresolved` and `@typescript-eslint/no-unsafe-*` errors
- [eslint/no-unnecessary-type-conversion]: Don't wrap strings in `String()` ‚Äî `@typescript-eslint/no-unnecessary-type-conversion` flags redundant conversions
- [eslint/no-unnecessary-condition]: When casting with `as number`, use `as number | undefined` if the value may be undefined ‚Äî otherwise `?? fallback` is flagged as unnecessary
- [eslint/no-unnecessary-condition]: Don't use `?.` on properties that are always present when the parent exists (e.g. `data?.pagination.total` not `data?.pagination?.total`)
- [testing]: When mocking `@laila/database`, include ALL imported functions (e.g. `writeAuditEvent`) ‚Äî missing mocked exports become `undefined` and cause runtime TypeErrors (500s)
- [testing]: When handler response shape changes, update test assertions to match ‚Äî check `data.task.status` / `data.cascading_updates.unblocked_tasks` not legacy shapes
- [query-hooks]: When adding new UI components that import hooks from `@/lib/query-hooks`, ensure the hooks are actually defined in that file ‚Äî missing exports cause cascading `@typescript-eslint/no-unsafe-*` errors on every usage site
- [schema]: When adding a new required field to an entity schema (e.g. `projectSchema`), update ALL test fixtures that construct instances of that entity ‚Äî including `project.test.ts`, `list-responses.test.ts`, and `crud-schemas.test.ts`
- [schema/api]: Fields with database defaults (e.g. `workerInactivityTimeoutMinutes`) should be `.optional()` in create schemas ‚Äî use `schema.shape.field.optional()` to keep validation but allow omission. Use conditional spread in the handler to avoid passing `undefined` to the repo
- [eslint/import-x/export]: Never define two `export const` with the same name in a file ‚Äî when refactoring hooks (e.g. from manual `fetch` to `apiClient`), remove the old version instead of leaving both
- [testing/schema-sync]: When adding a new required field to a Zod entity schema (e.g. `projectSchema`), update ALL test fixtures, mock factories (`createMockProject`), mock interfaces (`MockProject`), and request bodies that create or validate that entity ‚Äî the field propagates via `.omit()` / `.pick()` to create/update/response schemas
- [query-hooks/non-spec-endpoints]: When adding API pages for endpoints not in the OpenAPI spec (e.g. epic validate/publish/counts), also add the corresponding query hooks in `query-hooks.ts` using manual `fetch` ‚Äî components importing undefined hooks cause cascading `@typescript-eslint/no-unsafe-*` errors
- [query-hooks/completeness]: When adding validate/publish/counts UI flows for an entity, also add the corresponding `useValidate*`, `usePublish*`, `use*Counts` hooks to `query-hooks.ts` ‚Äî follow the raw `fetch` pattern used by story validate/publish hooks for endpoints not in the openapi-spec
- [query-hooks/dedup]: Before adding hooks to `query-hooks.ts`, search the file for existing exports with the same name ‚Äî multiple branches may independently add the same hooks (e.g. `useValidateEpic`, `usePublishEpic`, `useEpicCounts`), causing `import-x/export` duplicate-export errors
- [testing/db-mock]: When a handler does raw `db.select().from().innerJoin().where()` chains via `getDb()`, the test mock must return a chainable object (not `{}`), export mock table symbols, and mock `drizzle-orm` functions (`eq`, `and`, `isNull`, `notInArray`) so they don't throw on non-Column arguments
- [util/signature-sync]: When UI components destructure `{ formatted, risk }` from a utility, ensure the utility's return type matches ‚Äî if the function returns `string` but consumers expect an object with typed fields (e.g. `TimeoutRisk`), update the function signature and return type to match all call sites
